# 初回実装要求定義

## 概要

対戦型タイピングゲーム「タイピング道場」のMVP（実用最小限の製品）を実装する。

## 実装範囲

### 必須機能

1. **タイトル画面**
   - ゲームタイトル表示
   - 対戦開始ボタン
   - 操作説明（Caps Lock ON/OFF）

2. **ゲーム画面**
   - 上下分割レイアウト（Player 1: 上、Player 2: 下）
   - 出題文（日本語）とローマ字の表示
   - 入力済み文字のハイライト
   - リアルタイムスコア表示
   - 30秒カウントダウンタイマー

3. **プレイヤー分離**
   - Caps Lock OFF → Player 1（小文字入力）
   - Caps Lock ON → Player 2（大文字入力）

4. **入力処理**
   - ローマ字入力の正誤判定
   - 複数入力パターンの許容（し → si, shi, ci）
   - 拗音・促音の対応

5. **ゲーム進行**
   - 開始ボタン → カウントダウン（3, 2, 1）→ ゲーム開始
   - 1文打ち終わると次の文を表示
   - 30秒経過でゲーム終了

6. **結果表示**
   - 勝敗判定
   - 両者のスコア表示
   - もう一度対戦 / タイトルへ戻る ボタン

### 対象外（将来実装）

- 制限時間の選択機能
- API連携による出題
- スコア履歴
- 練習モード

## ユーザーストーリー

### US-001: ゲームを開始する

```
プレイヤーとして、
アプリを起動してすぐに対戦を始められるようにしたい。
なぜなら、準備に時間をかけずに遊びたいからだ。
```

**受け入れ条件:**
- `npm install` → `npm start` で起動できる
- タイトル画面が表示される
- 開始ボタンでゲームが始まる

### US-002: タイピングで対戦する

```
プレイヤーとして、
相手と同時にタイピングして競いたい。
なぜなら、リアルタイムの対戦を楽しみたいからだ。
```

**受け入れ条件:**
- 2人が同時に入力できる（Caps Lock で分離）
- 入力がリアルタイムで画面に反映される
- 正しい入力でスコアが加算される

### US-003: 勝敗を確認する

```
プレイヤーとして、
対戦終了後に結果を確認したい。
なぜなら、勝敗を明確に知りたいからだ。
```

**受け入れ条件:**
- 30秒経過で自動的に終了
- 勝者とスコアが表示される
- もう一度対戦できる

## 制約事項

- 出題文はひらがな・カタカナ・漢字のみ（記号・数字なし）
- 固定の出題リストから出題（最低10文）
- モダンブラウザ（Chrome, Firefox, Safari, Edge）対応

## 成功基準

- [ ] `npm install` → `npm start` で起動できる
- [ ] 2人が別々のキーボードで同時にタイピングできる
- [ ] 30秒の対戦が正常に完了する
- [ ] 勝敗が正しく判定される
- [ ] 全テストが通る
